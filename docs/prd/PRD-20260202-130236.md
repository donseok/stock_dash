# 주식 웹 포털 대시보드 PRD (Product Requirements Document)

**버전**: 1.0
**작성일**: 2026-02-02
**상태**: Draft

---

## 1. 개요

### 1.1 배경

현재 투자자들은 국내 주식, 해외 주식, 암호화폐, 귀금속, 환율 등 다양한 자산 정보를 확인하기 위해 증권사 앱, 거래소 사이트, 뉴스 포털 등 여러 플랫폼을 개별적으로 접속해야 합니다. 이러한 정보 분산은 시장 상황에 대한 통합적 판단을 어렵게 하고, 빠른 대응이 필요한 시점에 정보 탐색 시간을 증가시킵니다.

이를 해결하기 위해 국내외 주식, 암호화폐, 금/은, 환율, 시장지수, 뉴스를 하나의 웹 대시보드에서 실시간으로 통합 제공하는 포털 시스템을 구축합니다. 조회 전용 시스템으로서 매매 기능 없이 시세 모니터링과 정보 확인에 특화된 서비스를 목표로 합니다.

### 1.2 목표
- 국내외 주식 시세 및 차트를 실시간으로 표시하는 웹 대시보드 구축
- 금/은, 암호화폐 등 다양한 자산군의 시세 통합 제공
- 국내외 시장지수를 한눈에 파악할 수 있는 대시보드 뷰 제공
- 관련 뉴스를 실시간으로 수집·표시하여 투자 판단 지원
- 실시간 데이터 업데이트를 통한 지연 없는 시세 반영
- 종목을 유연하게 교체할 수 있는 위젯 기반 구조 제공

### 1.3 범위

#### 포함 범위
- 웹 기반 대시보드 포털 (주식 차트, 뉴스, 종목 정보 등)
- 국내주식 시세 표시 (에스피지, 에코프로, 셀트리온)
- 해외주식 시세 표시 (비트마인, 알파벳 C, 피그마)
- 암호화폐 시세 표시 (비트코인, 이더리움, 솔라나, XRP, 체인링크, 수이, 온도파이낸스)
- 금/은 시세 표시
- 환율 정보 표시 (USD/KRW 등)
- 국내 시장지수 (KOSPI, KOSDAQ)
- 해외 시장지수 (다우, 나스닥)
- 실시간 차트 (조회 전용)
- 뉴스 피드
- 종목 교체 가능 구조 (위젯 기반)

#### 범위 외
- 실제 매매/주문 기능 (트레이딩)
- 사용자 포트폴리오 관리 및 수익률 추적
- 모바일 네이티브 앱 개발
- 증권사 계좌 연동
- AI 기반 종목 추천 및 자동매매
- 채팅/커뮤니티 기능
- 세금 계산 및 신고 지원

### 1.4 대상 사용자

| 사용자 유형 | 역할 및 주요 활동 |
|------------|------------------|
| 개인 투자자 | 국내외 주식에 관심 있는 일반 투자자. 보유 종목과 관심 종목의 시세를 수시로 확인 |
| 멀티 자산 투자자 | 주식·귀금속·암호화폐 등 여러 자산군을 동시에 모니터링하는 투자자 |
| 데이트레이더 | 실시간 시장 동향과 뉴스를 빠르게 확인하여 단기 매매 의사결정에 활용 |
| 일반 사용자 | 시장 전반의 흐름을 개괄적으로 파악하려는 비전문 사용자 |

### 1.5 성공 지표
- 시세 데이터 실시간 갱신 지연시간 5초 이내
- 대시보드 초기 로딩 시간 3초 이내
- 일일 활성 사용자(DAU) 출시 3개월 내 1,000명 달성
- 사용자 평균 세션 체류시간 5분 이상
- 시스템 가용성(Uptime) 99.5% 이상 유지
- 표시 자산군 커버리지: 국내주식, 해외주식, 금/은, 환율, 암호화폐, 시장지수 전 항목 정상 제공

## 2. 기능 요구사항 (Functional Requirements)

### FR-001: 웹 대시보드 포털 구축
**우선순위**: HIGH
**설명**: 주식 차트, 뉴스, 종목 정보, 시장지수, 금/은, 환율, 암호화폐 시세를 위젯 형태로 배치하는 대시보드 웹 포털을 구축한다. 깔끔한 UI/UX를 지향한다.

**User Story**: 투자자로서, 나는 하나의 웹 페이지에서 모든 자산군의 시세와 뉴스를 확인하고 싶다, 여러 사이트를 오가는 불편함을 없애기 위해.

**Acceptance Criteria**:
- [ ] 웹 브라우저에서 대시보드에 접속할 수 있다
- [ ] 주식 차트, 뉴스, 종목 정보, 시장지수, 금/은, 환율, 암호화폐 위젯이 한 화면에 표시된다
- [ ] 반응형 레이아웃으로 다양한 화면 크기에 대응한다
- [ ] 깔끔하고 직관적인 UI 디자인이 적용된다

**출처**: 요구사항.txt (1~3행)

---

### FR-002: 실시간 주식 차트 표시
**우선순위**: HIGH
**설명**: 대시보드에 주식 차트를 표시하며, 차트 데이터는 실시간으로 업데이트된다. 차트는 조회 전용이며 매매 기능은 포함하지 않는다.

**User Story**: 투자자로서, 나는 실시간으로 갱신되는 주식 차트를 보고 싶다, 시세 변동을 즉시 파악하기 위해.

**Acceptance Criteria**:
- [ ] 선택한 종목의 차트가 대시보드에 표시된다
- [ ] 차트가 실시간으로 업데이트된다 (갱신 지연 5초 이내)
- [ ] 차트는 조회 전용이며, 매매/주문 기능이 없다
- [ ] 캔들차트, 라인차트 등 기본 차트 유형을 지원한다

**출처**: 요구사항.txt (3~4행, 11행)

### FR-012: 종목 교체 가능 구조
**우선순위**: HIGH
**설명**: 주식 및 암호화폐 종목을 유연하게 교체(추가/삭제)할 수 있는 위젯 기반 구조로 구성한다. 초기 셋팅 이후 종목 변경이 가능해야 한다.

**User Story**: 투자자로서, 나는 대시보드에 표시되는 종목을 자유롭게 변경하고 싶다, 관심 종목이 바뀔 때 쉽게 교체하기 위해.

**Acceptance Criteria**:
- [ ] 위젯 설정에서 종목을 추가/삭제/변경할 수 있다
- [ ] 종목 변경 시 해당 위젯의 데이터가 즉시 갱신된다
- [ ] 변경된 종목 구성이 저장되어 재접속 시 유지된다
- [ ] 주식과 암호화폐 모두 교체가 가능하다

## 3. 비기능 요구사항 (Non-Functional Requirements)

### NFR-001: 실시간 데이터 갱신 성능
**우선순위**: HIGH
**설명**: 외부 API로부터 수집한 시세 데이터가 5초 이내에 대시보드에 반영되어야 한다. WebSocket 기반 실시간 푸시를 통해 지연을 최소화한다.

**Acceptance Criteria**:
- [ ] 시세 갱신 지연시간이 5초 이내이다
- [ ] WebSocket 연결 끊김 시 자동 재연결된다

**출처**: 요구사항.txt (4행)

---

### NFR-002: 초기 로딩 성능
**우선순위**: HIGH
**설명**: 대시보드 초기 로딩 시간이 3초 이내여야 한다.

**Acceptance Criteria**:
- [ ] 최초 접속 시 화면이 3초 이내에 렌더링된다
- [ ] 주요 위젯 데이터가 점진적으로 로드된다 (스켈레톤 UI)

---

### NFR-003: 시스템 가용성
**우선순위**: HIGH
**설명**: 시스템 가용성(Uptime)을 99.5% 이상으로 유지한다.

**Acceptance Criteria**:
- [ ] 월간 다운타임이 3.6시간 이내이다
- [ ] 장애 발생 시 알림이 전달된다

---

### NFR-004: UI/UX 품질
**우선순위**: MEDIUM
**설명**: 깔끔하고 직관적인 디자인을 적용하여 사용자 편의성을 극대화한다.

**Acceptance Criteria**:
- [ ] 일관된 디자인 시스템이 적용된다
- [ ] 데이터 밀도가 높으면서도 가독성이 유지된다

## 4. 제약조건

### CON-002: 업비트 API 사용
**설명**: 암호화폐 실시간 데이터 수집에 업비트(Upbit) API를 사용한다. WebSocket 기반 실시간 시세 수신을 지원한다.

**출처**: 요구사항.txt (10행)

---

### CON-003: 조회 전용 시스템
**설명**: 차트는 조회용으로만 사용하며, 차트 내에서 직접 매매/주문 기능은 제공하지 않는다.

**출처**: 요구사항.txt (11행)

---

### CON-004: 무료 소프트웨어/기술스택 사용
**설명**: 기술스택은 무료(오픈소스) 소프트웨어만 사용하여 구축한다. 라이선스 비용이 발생하는 상용 소프트웨어는 사용하지 않는다.

**출처**: 요구사항.txt (13행)

## 5. 마일스톤

### M1: MVP - 핵심 대시보드 (Phase 1)
**목표**: 핵심 대시보드 프레임워크 및 실시간 데이터 기반 구축
**주요 산출물**:
- 대시보드 레이아웃 및 위젯 프레임워크
- 국내주식 시세 표시 (에스피지, 에코프로, 셀트리온)
- 암호화폐 시세 표시 (비트코인 외 6종)
- 실시간 차트 업데이트 (WebSocket 기반)
- 종목 교체 가능 구조 구현

### M2: 해외주식 및 추가 자산군 (Phase 2)
**목표**: 해외 자산 및 원자재/환율 시세 통합
**주요 산출물**:
- 해외주식 시세 표시 (비트마인, 알파벳 C, 피그마)
- 해외 시장지수 표시 (다우, 나스닥)
- 국내 시장지수 표시 (KOSPI, KOSDAQ)
- 금/은 시세 표시
- 환율 정보 표시

### M3: 뉴스 및 종합 기능 (Phase 3)
**목표**: 정보 통합 및 UX 완성
**주요 산출물**:
- 뉴스 피드 위젯
- 종목 상세정보 표시
- UI/UX 최적화 (깔끔한 디자인)

### M4: 안정화 및 배포 (Phase 4)
**목표**: 테스트, 최적화 및 운영 배포
**주요 산출물**:
- 통합 테스트 및 성능 최적화
- 운영 환경 배포
- 모니터링 구성
